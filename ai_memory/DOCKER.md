# Docker Конфигурация

Проект разворачивается через Docker Compose и состоит из трех основных сервисов.

## Сервисы

1. **bot**:
   - Образ: На основе `Dockerfile` (Python 3.11-slim).
   - Роль: Основное приложение (Telegram бот + сканеры).
   - Зависимости: `redis`, `postgres`.
   - Команда запуска: `python -m bot.main`.

2. **postgres**:
   - Образ: `postgres:16-alpine`.
   - Роль: Постоянное хранилище (сделки, алерты, история).
   - Данные: Объем `pg_data`.
   - БД: `arb_db` (пользователь `arb`).

3. **redis**:
   - Образ: `redis:7-alpine`.
   - Роль: Кэширование данных бирж, лимиты запросов.
   - Ограничение памяти: 64MB (LRU политика).

## Команды управления

- `docker-compose up -d` — запуск всех сервисов в фоне.
- `docker-compose logs -f bot` — просмотр логов бота.
- `docker-compose restart bot` — перезагрузка только бота.
- `docker-compose down` — остановка сервисов.
- `docker-compose down -v` — полная очистка (включая БД).

## Сетевое взаимодействие

- Сервисы связаны внутри Docker-сети.
- `DATABASE_URL` использует хост `postgres`.
- `REDIS_URL` использует хост `redis`.
